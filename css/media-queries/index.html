Le media-queries

← Torna al menù

Quando si vogliono realizzare pagine web che si adattano alle dimensioni dello schermo, in modo da realizzare siti web responsive, è necessario usare diversi fogli di stile per ciascuna dimensione per cui si vuole rendere disponibile.

Per fare ciò, è possibile utilizzare l'attributo media del tag like, così da poter specificare le dimensioni che la finestra dell'utente deve avere per importare quel determinato foglio CSS, nel seguente modo: 

<link rel="stylesheet" media="screen and (max-width: 599px)" href="small.css" type="text/css" />
<link rel="stylesheet" media="screen and (min-width: 600px) and (max-width: 1200px)" href="medium.css" type="text/css" />
<link rel="stylesheet" media="screen and (min-width: 1201px)" href="large.css" type="text/css" />

In questo modo, si specifica di importare il file small.css per gli schermi fino a 599px, medium.css per gli schermi compresi fra 600px e 1200px e large.css per gli schermi con dimensione sopra i 1200px.

Bisogna specificare che è possibile utilizzare come valore, oltre che min-width e max-width, anche min-device-width e max-device-width, che faranno quindi riferimento alle dimensioni dello schermo invece che alle dimensioni della finestra.

Media queries integrate

Se invece di creare diversi file CSS, si vuole scrivere un unico file che implementa tutte le configurazioni per le varie dimensioni, è possibile utilizzare le media-queries interne ai file CSS.

All'interno di un file CSS, per fare distinzioni riguardo le dimensioni della finestra, viene usata la seguente istruzione, che comprenderà all'interno delle parentesi tutto il codice da interpretare se la condizione è verificata. 

// file style.css
...

@media screen and (max-width: 599px) {
    ...
}
@media  screen and (min-width: 600px) and (max-width: 1200px) {
    ...
}
@media  screen and (min-width: 1201px) {
    ...
}

Questo codice avrà lo stesso identico risultato del codice precedente, con l'unica differenza che, nel codice HTML, sarò necessario inserire un solo link (e quindi caricare un unico file CSS, che verrà memorizzato in cache); nel codice CSS, invece, sarà necessario scrivere più codice, che renderà quindi il file CSS più pesante (che potrebbe non essere un problema, se la cache è attiva).
